- 现代 C++ 内存与值类别体系
  - 1. 左值与右值 (核心定义)
    - 左值 (Lvalue): 容器本体。有确定内存地址，有名字，能活过当前行，能用 `&` 取地址。(例: `++x`, 具名变量)
    - 右值 (Rvalue): 容器内容。无固定地址，无名字，用完即毁的临时数据，绝不能用 `&` 取地址。(例: `x++`, 临时拷贝, 字面量)
  - 2. 引用机制与致命易错点
    - 左值引用 (`&`): 给“活人”起别名。
    - 右值引用 (`&&`): 给“将死之物”强行续命，打上“可抢劫”标签。
    - 【致命易错点】: 右值引用变量本身是左值！(一进函数门，有了名字和栈地址，立刻受左值保护)
  - 3. 移动语义 (零开销性能引擎)
    - 本质: 避免昂贵的深拷贝，直接“趁火打劫”窃取底层堆内存指针。
    - 核心实现 2 步走: 1. 偷走指针 -> 2. 原主人指针置空 (防 Double Free 炸内存)。
    - 双重视角 (架构哲学):
      - 门外 (调用方视角): 传进去的是右值 -> 决定了编译器路由到 `&&` 移动车道。
      - 门内 (被调方视角): 变量本身变成左值 -> 赋予开发者合法操作/置空的物理实体权限。
    - `std::move` 的本质:
      - 事实: 一行汇编不产生，不移动任何东西。
      - 机制: 强行将左值转换为将亡值 (xvalue)，相当于签了“放弃急救同意书”。
      - 【易错点】: 勿对 `const` 对象 move (退化为深拷贝)；move 后勿再访问原对象 (已掏空)。
  - 4. 完美转发 (打破中间商吃差价)
    - 业务痛点: 代理函数传参时，参数具名化变为左值，导致底层永远错误触发深拷贝。
    - 破局基石 1: 万能引用 (`T&&`) -> 配合模板推导的变色龙，能接左值也能接右值。
    - 破局基石 2: 引用折叠规则 -> 依靠 `&` 强势基因保留原始身份 (仅 `&& + && = &&` 保留右值属性)。
    - 终极杀招: `std::forward<T>()` -> 条件类型转换。查模板底单，原是左值不动，原是右值则套 `std::move` 打回原形。
  - 5. 值类别终极矩阵 (Bjarne Stroustrup 终极分类法)
    - X 轴 (身份属性): 有名字 (Identity) vs 没名字
    - Y 轴 (权限属性): 不能移动 vs 能移动 (Movability)
    - Lvalue (左上): 有名字 + 不能移动 (普通变量，受绝对保护)
    - Prvalue (右下): 没名字 + 能移动 (临时对象，天生炮灰)
    - Xvalue (左下): 有名字 + 能移动 (主动用 `std::move` 献祭的将亡值)
