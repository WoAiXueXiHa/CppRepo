# 裸指针与底层架构

## 1. 物理映射与底层机制
### 1.1 本质定义
- **普通变量**：内存单间的别名 (存放业务数据)
- **指针变量**：独立的内存单间，专门存放目标数据首地址的变量 (64位固定占8字节)
### 1.2 怎么使用 (易错点)
- `&` 取址符：获取单间物理门牌号
- `*` 解引用：按门牌号推开门操作数据 (声明时的 `*` 与操作时的 `*` 意义完全不同)


## 2. 步长管理与形态 (核心基石)
### 2.1 本质定义
- 指针类型决定跨越步长和解析方式
### 2.2 怎么使用 (易错点)
- **步长公式**：`p + 1` 物理偏移量 = `sizeof(指针类型)`
- **内置步长**：`char*`(1字节), `int*`(4字节), `double*`(8字节)
- **void* 陷阱**：无步长万能指针，严禁直接算术运算或解引用，必须先强转恢复形态。
- **强制转换陷阱**：指针转为整数 (`unsigned long`) 后彻底失去指针属性，`+1` 仅前进 1 字节；极易受操作系统大小端 (Endianness) 影响。

## 3. 数组与指针的降维恩怨
### 3.1 本质定义
- **数组**：连续分配的物理实体仓库 (自带总长度信息)
- **指针**：独立存放地址的单间
### 3.2 怎么使用 (易错点)
- **数组退化 (Array Decay)**：多数表达式中，数组名自动退化为指向首元素的裸指针 (`&arr[0]`)。
- **不退化特例 (必考)**：
  1. `sizeof(arr)`：求整个实体总字节大小。
  2. `&arr`：求实体首地址，此时指针步长变为**整个数组的总大小**。
- **传参致命伤**：数组作为函数入参必定退化为指针，函数内部 `sizeof` 无法获取正确长度，务必同时传入长度参数。

## 4. 高阶指针与优先级博弈
### 4.1 本质定义
- 结合法则：`[]` (数组修饰) 优先级高于 `*` (指针修饰)。看变量名先跟谁结合，本体就是谁。
### 4.2 怎么使用 (易错点)
- **指针数组 (`int* arr[10]`)**：本体是数组，货架上存满指针。极易忘记给内部元素逐个 `new` 分配内存导致野指针。
- **数组指针 (`int (*p)[10]`)**：本体是指针，指向含10个元素的连续数组。`p + 1` 会直接跨越“一整行”的物理内存 (`10 * 4 = 40` 字节)。

## 5. 函数与指针的极致解耦
### 5.1 本质定义
- **指针函数 (`int* func()`)**：返回内存地址的普通工厂函数。
- **函数指针 (`int (*p)()`)**：存放代码段中机器指令入口地址的变量。
### 5.2 怎么使用 (易错点)
- **指针函数致命死穴**：严禁返回局部栈变量的地址 (出栈即销毁，直接造就悬空指针)。
- **函数指针规范**：必须使用 `using` 或 `typedef` 简化签名；小心 Lambda 异步回调时的引用捕获 (`[&]`) 生命周期灾难。
### 5.3 价值所在
- 实现**回调机制 (Callback)**，是底层网络框架 (机制) 与上层业务逻辑 (策略) 彻底解耦的绝对基石 (如 Reactor 模型)。

## 6. 异常指针与内核防线
### 6.1 本质定义
- **空指针 (Nullptr)**：受控炸弹，指向被 OS 物理隔离的 `0x0` 禁区。
- **野指针 (Wild)**：隐形杀手，未初始化的随机指针。
- **悬空指针 (Dangling)**：核心死敌，内存已释放/销毁但指针未置空 (拿旧钥匙开拆迁房)。
### 6.2 怎么使用 (易错点)
- **内核审判链路**：解引用空指针 -> MMU硬件缺页异常 -> 内核发送 SIGSEGV -> 进程暴毙 (Core Dump)。
- **工程防御**：声明即初始化；用前必须判空；`delete` 后立刻置为 `nullptr`。

## 7. const 契约与对齐压榨
### 7.1 本质定义
- **const边界**：视 `*` 为墙。
- **内存对齐**：配合 CPU 按字长块状寻址，编译器自动填补 Padding (空气) 消除二次读取开销。
### 7.2 怎么使用 (易错点)
- **左定值 (`const int* p`)**：锁定货物数据，指针可换指向 (最常用：只读入参契约)。
- **右定向 (`int* const p`)**：锁定方向盘，指向焊死不可变，数据可改 (硬件映射)。
- **对齐陷阱**：结构体大小通常大于成员总和。网络协议栈序列化必须 `#pragma pack(1)` 取消对齐。
